# FullVid Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A Playnite plugin to watch YouTube gameplay/trailer videos fullscreen with controller navigation, and download them into the ExtraMetadataLoader (EML) folder as a complement to EML.

**Architecture:** Port UniPlaySong's proven controller-input router + dialog helper + Material dark theme; add a WebView2 IFrame-embed video surface driven from C#, a yt-dlp structured-search service, and a yt-dlp+ffmpeg download→transcode pipeline that writes EML's `VideoTrailer.mp4` contract. Pauses UniPlaySong music via UPS's existing `playnite://uniplaysong/pause` URI while watching.

**Tech Stack:** Playnite SDK 6.16.0.0, .NET Framework 4.6.2, WPF (`UseWPF`), AnyCPU (32-bit host), MaterialDesignThemes 4.7.0 + MaterialDesignColors 2.1.0, `Microsoft.Web.WebView2` (x86 runtime), Newtonsoft.Json 13.0.1, yt-dlp + ffmpeg (user-installed paths). NUnit + Moq for tests.

## Global Constraints

- **Reuse source lives at `..\UniPlaySong\src\`** — copy these files near-verbatim (rename namespace `UniPlaySong.*` → `FullVid.*`): `Services/Controller/ControllerEventRouter.cs`, `Services/Controller/IControllerInputReceiver.cs`, and the reusable parts of `Common/DialogHelper.cs`. DO NOT reuse `Downloaders/YouTubeClient.cs` (HTML scraper).
- **yt-dlp is the search + download engine.** Verified command (yt-dlp 2025.11.12): `yt-dlp "ytsearch{N}:{query}" --dump-json --flat-playlist --no-warnings` → one JSON object per line with fields: `id` (string), `title` (string), `duration` (float seconds), `channel`/`uploader` (string), `view_count` (int), `thumbnails` (array; `[0].url` is a direct `i.ytimg.com` URL), `webpage_url` (string). Always run `--version` against the user's configured binary before use (never assume flags — verify-cli-flags rule).
- **EML folder contract (verified vs EML source 12c75d4):** write `{PlayniteApi.Paths.ConfigurationPath}\ExtraMetadata\games\{game.Id}\VideoTrailer.mp4`. Casing exact: `ExtraMetadata` (Pascal), `games` (lower). `{game.Id}` = `game.Id.ToString()` (GUID, no braces). Anchor on `ConfigurationPath` — NEVER hardcode `%AppData%\Playnite`.
- **EML-compatible video format:** MP4, H.264 (`-c:v libx264`), `-pix_fmt yuv420p`, AAC audio (`-c:a aac`), even dimensions (`-vf scale=trunc(iw/2)*2:trunc(ih/2)*2`). EML plays via WPF MediaElement (WMF) — VP9/WebM/Opus won't play.
- **Controller UX:** all transport stays in C# (via the router), NEVER delegated to the web page. B = close (instant), A = confirm/play-pause, respect Playnite `SwapConfirmCancelButtons`.
- **Commit trailer:** `Co-Authored-By: Claude` — never a model name, no email.
- **Build gate every task:** `dotnet build -c Release` succeeds.
